# AI Risk Checklist

## 1. 用例适当性

- [ ] 是否有明确业务目标？
- [ ] 是否真的需要 AI？
- [ ] 是否可以用规则、流程或传统系统解决？
- [ ] 是否涉及高影响决策？

## 2. 数据

- [ ] 数据来源是否合法？
- [ ] 是否包含敏感个人信息？
- [ ] 是否有数据最小化原则？
- [ ] 是否有数据质量评估？
- [ ] 是否有数据血缘和留痕？

## 3. 模型

- [ ] 模型能力是否适配场景？
- [ ] 是否有基准测试？
- [ ] 是否有可解释性要求？
- [ ] 是否评估稳定性和鲁棒性？
- [ ] 是否有模型版本管理？

## 4. 大模型特殊风险

- [ ] 是否评估幻觉？
- [ ] 是否防止提示注入？
- [ ] 是否限制敏感信息输出？
- [ ] 是否有引用来源？
- [ ] 是否有人工复核？

## 5. 第三方风险

- [ ] 是否使用外部模型或云服务？
- [ ] 是否评估供应商风险？
- [ ] 是否明确数据不用于训练？
- [ ] 是否有服务可用性和退出方案？

## 6. 上线后监控

- [ ] 是否监控准确性？
- [ ] 是否监控异常输出？
- [ ] 是否监控用户反馈？
- [ ] 是否定期复评？
- [ ] 是否有停用机制？
