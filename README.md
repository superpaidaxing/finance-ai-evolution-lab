# Finance AI Evolution Lab

> 一个面向银行从业者的“金融 + AI”长期进化式学习与实践项目。目标不是追热点，而是把 18 年银行业务经验转化为可复利、可沉淀、可验证、可公开迭代的知识与作品体系。

## 1. 项目定位

本项目用于长期研究和实践：

- AI 如何重塑银行、金融服务、风险管理、合规、运营、客户经营与投研。
- 银行业务经验如何转化为 AI 时代的业务洞察、产品能力、数据能力与治理能力。
- 如何通过 GitHub 形成一个可持续更新的“个人金融 AI 操作系统”。

适合长期目标：

1. 建立金融 AI 的系统知识地图。
2. 每周持续沉淀阅读笔记、案例分析、实验记录和复盘。
3. 把银行实务场景抽象成 AI 可落地的问题。
4. 从“理解 AI”进阶为“能设计、能验证、能治理、能表达金融 AI 方案”。
5. 形成长期可展示的公开作品集。

## 2. 核心原则

### 2.1 业务经验是护城河

18 年银行经验不是负担，而是最大优势。金融 AI 的深度不只来自模型算法，更来自：

- 对客户、产品、流程、风险、监管和组织运行的真实理解。
- 对银行业务边界、合规红线和风险成本的敏感度。
- 对“一个 AI 用例为什么能落地 / 为什么不能落地”的判断力。

### 2.2 每个主题都要经过四层闭环

每个学习主题都按同一套闭环推进：

1. **概念层**：这个概念是什么？来自金融还是 AI？
2. **业务层**：它在银行真实场景中解决什么问题？
3. **技术层**：需要哪些数据、模型、系统和评估方法？
4. **治理层**：有什么风险、合规、伦理、模型管理和组织落地问题？

### 2.3 少追热点，多做复利

每周只要完成一个小更新也可以，但必须有沉淀：

- 一篇阅读笔记
- 一个业务案例拆解
- 一个 AI 工具实验
- 一个 Prompt / RAG / Agent 原型
- 一个风险治理清单
- 一个阶段复盘

## 3. 推荐长期研究主线

### A. 银行业务 + AI 场景主线

- 零售银行：客户画像、智能营销、财富顾问、信贷审批、贷后管理。
- 公司银行：授信辅助、企业画像、票据/供应链金融、现金管理。
- 风险管理：信用风险、市场风险、操作风险、流动性风险、压力测试。
- 合规与反洗钱：KYC、AML、交易监测、可疑行为解释、监管报告。
- 运营与客服：智能客服、投诉处理、流程自动化、知识库助手。
- 管理驾驶舱：经营分析、风险预警、会议纪要、制度问答。

### B. AI 能力主线

- Python、数据分析、统计学与机器学习基础。
- 金融机器学习：评分卡、风控模型、时间序列、异常检测。
- 大语言模型：Prompt Engineering、RAG、Agent、工具调用、多模态。
- 评估体系：准确率、召回率、稳定性、幻觉率、可解释性、业务收益。
- AI 工程：数据管道、向量数据库、API、自动化工作流、实验记录。

### C. 治理与监管主线

- 模型风险管理
- 数据隐私与安全
- 第三方模型和云服务风险
- 可解释性、公平性、稳健性
- AI 项目立项、审批、上线、监控和退役机制
- Basel、NIST AI RMF、金融监管与行业报告跟踪

### D. 个人输出主线

- 每月一篇深度文章
- 每季度一个小型实验项目
- 每半年一个体系化专题
- 每年一次年度金融 AI 趋势复盘

## 6. GitHub 在线迭代方式

你可以完全在 GitHub 网页端维护，不需要本地双向同步：

1. 打开仓库文件。
2. 点击右上角铅笔图标编辑。
3. 修改 Markdown 内容。
4. 提交 Commit。
5. 用 Issue 管理待学习主题。
6. 用 GitHub Projects 管理长期路线图。
7. 每周固定做一次 `00-meta/weekly-review.md` 复盘。

建议建立 GitHub Project 字段：

- Status：Backlog / This Week / In Progress / Done / Archived
- Track：Finance / AI / Use Case / Governance / Lab / Output
- Depth：L1 入门 / L2 能解释 / L3 能设计 / L4 能验证 / L5 能输出
- Quarter：2026-Q2、2026-Q3 等

如果坚持 1 年，这个仓库应当能回答：

- 我对金融 AI 的理解是否系统化？
- 我是否能把银行场景拆成 AI 问题？
