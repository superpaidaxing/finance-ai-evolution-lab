# 11 开源金融 AI 项目库

这里专门收集公开开源的“金融 + AI”项目，用于后续学习、拆解、复现和改造成自己的小实验。

## 收录原则

1. **必须能公开访问**：优先收录 GitHub 等公开仓库。
2. **必须有明确金融场景**：包括银行、投研、量化交易、风险、反欺诈、财报分析、监管文件分析、金融客服等。
3. **必须有 AI 能力**：包括大语言模型、检索增强生成、智能体、机器学习、深度学习、强化学习、图神经网络、异常检测等。
4. **优先收录仍在维护的项目**：优先选择 2024-2026 年仍有更新、发布或活跃讨论的项目；少量经典项目会单独标注“偏经典/需二次维护”。
5. **不收录闭源产品**：商业 SaaS、只提供演示但没有源码的项目不放入主库。
6. **不把项目介绍写成广告**：重点解释它能学什么、能迁移到银行什么场景、有哪些限制和风险。

## 去重规则

后续分批追加时，按以下字段同时去重：

- 项目名称
- GitHub 仓库地址
- 所属组织或作者
- 核心功能定位

如果一个项目有多个仓库，只把“主仓库”作为项目条目；相关子仓库只在说明中引用，避免重复计数。

## 建议阅读方法

不要一口气把所有项目都下载运行。建议按用途分层阅读：

1. **先看项目定位**：它到底解决什么金融问题。
2. **再看核心 AI 能力**：是大模型、RAG、智能体、机器学习、强化学习，还是图模型。
3. **再看能否迁移到银行**：能不能迁移到客服、合规、信贷、投研、风控、运营、管理分析。
4. **最后再决定是否复现**：只挑 1-2 个项目做小实验，避免陷入环境配置和工具堆叠。

## 推荐优先级

对银行从业者来说，建议优先顺序是：

1. 金融文档问答与分析类：更贴近制度、报告、研报、财报、监管文件。
2. 银行 RAG / 客服助手类：更容易迁移到知识库问答、客户经理助手、运营支持。
3. 风控反欺诈类：适合理解图模型、异常检测、交易风险识别。
4. 金融大模型与智能体类：适合理解未来投研、分析、自动化工作流。
5. 量化交易类：技术价值高，但与传统商业银行岗位的直接关联相对弱，适合作为 AI 方法学习材料。
